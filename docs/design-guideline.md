# seulki.log 디자인 가이드라인

값을 새로 만들지 않는다. 아래 스케일에서만 고른다.
새 값이 필요하다고 느끼면, 먼저 "기존 스케일 중 하나로 안 되나"를 확인한다.

---

## 1. 세로 간격 (가장 자주 틀리던 곳)

CSS 변수 4개만 쓴다. 임의값(`1.3rem`, `3rem` 같은 것) 금지.

| 변수 | 값 | 쓰는 곳 |
|---|---|---|
| `--sp-section` | `clamp(2.4rem, 6vw, 3.6rem)` | 큰 경계: 헤더↔본문↔푸터 |
| `--sp-block` | `2.4rem` | 블록 사이: 본문↔반응바, 모바일 스택 사이 |
| `--sp-tight` | `1.4rem` | 붙는 요소: 마스트헤드 안쪽, 푸터 내부 묶음 |
| `--sp-hair` | `.6rem` | 최소 간격 |

**핵심 규칙: 경계의 위·아래는 항상 같은 값.**
헤더 아래가 `--sp-section`이면 푸터 위도 `--sp-section`이다. 지금까지 문제의 대부분이 이 비대칭(위 3.6rem / 아래 1rem)에서 나왔다.

적용 지점:
- `.masthead` 안쪽 위아래 = `--sp-tight`
- `.post` 위 = `--sp-section` / `main` 아래 = `--sp-section` (한 쌍)
- `.footin` 위아래 = `--sp-section`
- `.reactions` 위 = `--sp-block`

| 크기 | 쓰는 곳 |
|---|---|

| `.7rem` | 글목록 카테고리 · 날짜 |
| `.72rem` | 메타 · 버튼(DARK/메일) · 이탤릭 · 저작권 |
| `.83rem` | 글목록 제목 · 레일 로고 · 메뉴 · 사이트 소개문 |

본문·제목은 `calc(… * var(--reader))`로 감싼다(독자 글자크기 조절 지원).

## 4. 색

Color Hunt 뉴트럴. 라이트/다크 모두 변수로만 쓴다.

| 변수 | 라이트 | 쓰임 |
|---|---|---|
| `--paper` | `#DCD6CA` | 배경 |
| `--band` | `#D0CABD` | 선택된 행 배경 |
| `--ink` | `#222831` | 본문 |
| `--ink-soft` | `#393E46` | 보조 텍스트 |
| `--ink-faint` | `#6E6656` | 메타 |
| `--line` | `#C2BCAD` | **구분선(보이는 선)** |
| `--line-soft` | `#D0CABC` | 아주 약한 선 |
| `--accent` | `#948979` | 채움 전용(점·진행바). 본문 링크색 아님 |

주의: `--line-soft`는 `--band`와 거의 같아 **선택된 행 위에서는 안 보인다.** 글목록 구분선은 반드시 `--line`.

## 5. 형태 · 강조

- **각진 네모**: `border-radius: 0`. 예외는 선택 표시 점(원형) 하나.
- 강조는 **굵기 · 배경 · 밑줄**로만. 좌측 accent 바·장식 바 금지.
- **구분선 두께는 `0.5px`** (헤어라인). `1px`는 두꺼워 보인다.
- 아이콘 링크(인스타·메일)는 **아이콘만**. 텍스트·화살표·밑줄·테두리 없음.
  - 반응바 안에 넣을 때는 `.reactions button`의 공통 테두리를 반드시 해제할 것.

### 아이콘 규칙

- **선 굵기 `1.2`** — 주변 글자(400)와 무게를 맞춘다.
- **모서리 각지게** — 아이콘 안 `rect`도 `rx: 0`. 인스타 기본 글리프는 둥근 모서리라 반드시 해제.
- **마크업이 아니라 CSS로 통제**한다. `.ci { stroke-width }` / `.ci rect { rx }` 한 곳만 고치면 세 군데(반응바·푸터·홈 레일)에 동시 적용된다. SVG 속성보다 CSS가 우선.
- **크기를 옆 요소와 맞출 때는 테두리를 덧씌우지 말고**, 총 높이(패딩+글리프)를 같게 하고 글리프를 키운다.
  - 예: 반응바에서 `링크 복사` 버튼 높이 ≈ 21px → 아이콘 `16px 글리프 + .16rem 패딩`

### 연락 아이콘 위치 (고정)

| 페이지 | 위치 |
|---|---|
| 글(posts) | 반응바 **링크 복사 옆** |
| 소개·목록 등 그 외 | **푸터 맨 아래** (저작권 위) |
| 홈 | 왼쪽 레일 하단 |

한 페이지에 두 곳 이상 노출하지 않는다. 메일 모달(`dialog`)은 조건문 밖에 두어 모든 페이지에서 열리게 한다.

## 6. 애니메이션

| 이름 | 용도 | 시간 |
|---|---|---|
| `ix-blink` | 선택된 글 표시 점 | 1.4s |
| `ix-marquee` | 잘린 제목 흐르기 (데스크톱 호버) | 7.5s |
| `ix-marquee` | 잘린 제목 자동 (터치 기기) | 5s |
| `panel-in` | 홈 오른쪽 패널 전환 페이드 | .35s |

`prefers-reduced-motion: reduce`에서는 **모든 애니메이션·트랜지션이 꺼진다.** 새 애니메이션을 넣어도 이 규칙은 유지할 것.

---

## 7. 작업 순서 (여백 문제 재발 방지)

1. 새 여백이 필요하면 **먼저 스케일에서 고른다.**
2. 경계를 만들 땐 **위·아래를 같은 변수로** 넣는다.
3. 값을 하드코딩했다면 그건 버그다. 변수로 바꾼다.
