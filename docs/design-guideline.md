# seulki.log 디자인 가이드라인

값을 새로 만들지 않는다. 아래 스케일에서만 고른다.
새 값이 필요하다고 느끼면 먼저 "기존 스케일 중 하나로 안 되나"를 확인한다.

**한 줄 원칙: 하드코딩된 값은 버그다.**

---

## 1. 기반

| 항목 | 값 |
|---|---|
| 서체 | `Pretendard Variable` (셀프호스팅) → Pretendard → -apple-system → Apple SD Gothic Neo |
| 본문 기준 크기 | `1.0625rem` (17px) × `--reader` |
| 독자 글자크기 | `--reader` = `.92` / `1` / `1.12` (A A A 버튼, localStorage 저장) |
| 기본 자간 | `--tracking: -0.015em` (자간 -15) |
| 읽기 폭 | `--measure: 42rem` |
| 좌우 여백 | `--mx: clamp(1.4rem, 5vw, 2.2rem)` |

본문·제목 크기는 반드시 `calc(… * var(--reader))`로 감싼다. 안 그러면 독자 글자크기 조절이 안 먹는다.

---

## 2. 세로 간격 (가장 자주 틀리던 곳)

CSS 변수 4개만 쓴다. 임의값(`1.3rem`, `3rem` 같은 것) 금지.

| 변수 | 값 | 쓰는 곳 |
|---|---|---|
| `--sp-section` | `clamp(2.4rem, 6vw, 3.6rem)` | 큰 경계: 헤더↔본문↔푸터 |
| `--sp-block` | `2.4rem` | 블록 사이: 본문↔반응바, 모바일 스택 |
| `--sp-tight` | `1.4rem` | 붙는 요소: 마스트헤드 안쪽, 푸터 내부 |
| `--sp-hair` | `.6rem` | 최소 간격 |

**핵심 규칙: 경계의 위·아래는 항상 같은 값.**
헤더 아래가 `--sp-section`이면 푸터 위도 `--sp-section`이다. 지금까지 여백 문제의 대부분이 이 비대칭(위 3.6rem / 아래 1rem)에서 나왔다.

적용 지점: `.masthead` 안쪽 = `tight` · `.post` 위 / `main` 아래 = `section`(한 쌍) · `.footin` 위아래 = `section` · `.reactions` 위 = `block`

## 6. 색

Color Hunt 뉴트럴. **반드시 변수로만 쓴다.** 하드코딩된 헥사값 금지(에러 색 하나만 예외).

| 변수 | 라이트 | 다크 | 쓰임 |
|---|---|---|---|
| `--paper` | `#DCD6CA` | `#222831` | 배경 |
| `--band` | `#D0CABD` | `#393E46` | 선택된 행 배경 |
| `--card` | `#E4DFD5` | `#2E343D` | 호버 배경 |
| `--ink` | `#222831` | `#DFD0B8` | 본문·제목 |
| `--ink-soft` | `#393E46` | `#B7AE96` | 보조 텍스트 |
| `--ink-faint` | `#6E6656` | `#847B6B` | 메타·날짜·저작권 |
| `--line` | `#C2BCAD` | `#3D434C` | **보이는 구분선** |
| `--line-soft` | `#D0CABC` | `#31363E` | 아주 약한 선 |
| `--accent` | `#948979` | `#A99C84` | 채움 전용(점·진행바·아바타) |

### 색 함정 (실제로 겪음)

- **`--line-soft`는 `--band`와 거의 같다** (`#D0CABC` vs `#D0CABD`). 선택된 행 위에서는 안 보인다. **글목록 구분선은 반드시 `--line`.**
- **`--accent`는 본문 링크색이 아니다.** 채움(배경·점·바)에만 쓴다. 링크는 `--ink` + 밑줄.
- 다크 모드는 단순 반전이 아니다. 새 색을 넣으면 `:root[data-theme="dark"]`와 `prefers-color-scheme: dark` **양쪽 모두에** 추가해야 한다(두 곳에 중복 정의되어 있음).

## 7. 레이아웃 · 반응형

| 브레이크포인트 | 의미 |
|---|---|
| `> 820px` | 데스크톱. 홈은 3열(레일 / 글목록 / 본문) |
| `≤ 820px` | 홈이 1열 세로 스택으로 전환 |
| `≤ 560px` | 좁은 폰. 글목록 카테고리 폭 축소 |
| `(hover: none)` | 터치 기기. 호버 대신 자동 동작 |
| `(prefers-reduced-motion)` | 애니메이션 전부 정지 |

- 본문 페이지는 `.wrap`(`max-width: --measure`, `margin: 0 auto`)으로 가운데 정렬.
- **`.wrap`인 요소에 `margin-left`를 주지 말 것.** `margin: 0 auto`를 덮어써서 왼쪽으로 쏠린다(실제로 겪은 버그). 안쪽 자식에 주면 안전하다.

## 8. 형태 · 강조

- **각진 네모**: `border-radius: 0`. 예외는 선택 표시 점(원형) 하나.
- 강조는 **굵기 · 배경 · 밑줄**로만. 좌측 accent 바·장식 바 금지.
- **구분선 두께는 `0.5px`** (헤어라인). `1px`는 두꺼워 보인다.

## 9. 아이콘

- **마크업 단일 소스**: `layouts/partials/contact-icons.html`. 푸터·홈 레일이 모두 이 파샬을 부른다. 고칠 일이 생기면 여기 한 곳만.
- **굵기·모서리는 CSS로 통제**: `.ci { stroke-width: 1.2 }` / `.ci rect { rx: 0 }`. SVG 속성보다 CSS가 우선.
- **두 글리프는 폭 19로 통일**해 크기가 같아 보이게 한다 (인스타 19×19, 메일 19×15 봉투).
- 아이콘 링크는 **아이콘만**. 텍스트·화살표·밑줄·테두리 없음.
- 크기를 옆 요소와 맞출 때 **테두리를 덧씌우지 말고**, 총 높이(패딩+글리프)를 같게 하고 글리프를 키운다.
- 버튼 공통 스타일이 있는 영역(예: `.reactions button`)에 넣을 때는 그 테두리를 반드시 해제한다.

### 연락 아이콘 위치 (고정)

**모든 페이지에서 같은 자리** — 푸터 안, 작가소개 아래, 저작권 위, **왼쪽 정렬**.
홈만 예외로 왼쪽 레일 하단. 한 페이지에 두 곳 이상 노출하지 않는다.
메일 모달(`dialog`)은 조건문 밖에 두어 모든 페이지에서 열리게 한다.

## 10. 애니메이션

| 이름 | 용도 | 시간 |
|---|---|---|
| `ix-blink` | 선택된 글 표시 점 | 1.4s |
| `ix-marquee` | 잘린 제목 흐르기 (데스크톱 호버) | 7.5s |
| `ix-marquee` | 잘린 제목 자동 (터치 기기) | 5s |
| `panel-in` | 홈 오른쪽 패널 전환 페이드 | .35s |
| 읽기 진행바 | 글 페이지 상단 2px | 스크롤 비례 |
| 맨 위로 | 400px 넘으면 페이드 인 | .25s |

`prefers-reduced-motion: reduce`에서 **모든 애니메이션·트랜지션이 꺼진다.** 새 애니메이션을 넣어도 이 규칙은 유지할 것.

## 11. 자산 위치 (캐시버스팅)

CSS·JS는 `static/`이 아니라 **`assets/`** 에 있다.

- `assets/css/style.css` · `assets/js/site.js` · `assets/js/reactions.js`
- 템플릿에서 `resources.Get "css/style.css" | fingerprint` 로 불러온다.
- 빌드 시 파일명에 콘텐츠 해시가 붙어(`style.<hash>.css`) **배포하면 브라우저가 즉시 새 파일을 받는다.** 강력 새로고침 불필요.
- CSS 안 폰트 경로 `url("../fonts/…")`는 출력이 `/css/` 아래 유지되므로 그대로 동작한다.

`static/`에는 폰트·이미지·CNAME·robots.txt만 둔다.

---

## 12. 접근성

- 아이콘만 있는 버튼·링크에는 **반드시 `aria-label`과 `title`**.
- `:focus-visible`은 `2px solid var(--ink)` 아웃라인 유지.
- 색만으로 정보를 전달하지 않는다(선택 표시는 점 + 배경 두 가지로).
- `prefers-reduced-motion` 존중.

## 13. 작업 순서 (재발 방지)

1. 새 값이 필요하면 **먼저 스케일에서 고른다.**
2. 경계를 만들 땐 **위·아래를 같은 변수로** 넣는다.
3. 같은 마크업을 두 번째로 복사하게 되면 **파샬로 뽑는다.**
4. 값을 하드코딩했다면 그건 버그다. 변수로 바꾼다.
5. 배포 후에는 **라이브에서 실제로 반영됐는지 확인**하고 끝낸다.
